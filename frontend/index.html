How the Classification Pipeline Works

Each uploaded bank statement goes through a 4-stage pipeline:

  1. PDF Extraction — The bank profile parser extracts raw text, headers, and transaction lines from the PDF
  2. Data Cleansing — Duplicates are removed, amounts validated, and data is stored in the database
  3. Regex Classification — Each transaction is matched against the regex rules in priority order. First match wins.
  4. AI Classification — Remaining unclassified transactions are sent to Claude AI, which classifies them and proposes new regex patterns for human review

Improving Classification Accuracy

Follow this workflow to train the system and improve results over time:

  1. Process a statement — Upload a PDF on the Dashboard or Jobs page
  2. Review the results — In the Job Detail view, check each transaction's category and classification method
  3. Identify misclassifications — Look for wrong categories, especially AI-classified lines with low confidence
  4. Create or fix rules — Go to the Rules page to add, edit, or reorder regex patterns
  5. Review AI suggestions — Check the Refinements page for AI-proposed patterns and approve or reject them
  6. Reprocess — After adding/fixing rules, reprocess the statement to see improved results
  7. Repeat — Each iteration makes the regex rules more comprehensive, reducing AI dependency
